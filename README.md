# API Explorer

Это приложение позволяет взаимодействовать с JSONPlaceholder API, предоставляя удобный интерфейс для поиска информации о постах, комментариях, фотографиях и пользователях.

## Особенности

*   **Выбор раздела:** Пользователь может выбрать, с каким разделом API он хочет работать (посты, комментарии, пользователи).
*   **Поиск по параметрам:** Предоставляется возможность искать данные по различным параметрам (ID, ID автора, username и т.д.).
*   **Обработка ошибок:** Реализована обработка клиентских и серверных ошибок, а также пользовательских исключений.
*   **Логирование:** Все действия пользователя и ошибки записываются в файл `logs.txt`.
*   **Чистый интерфейс:** Консоль очищается перед каждым новым шагом взаимодействия.

## Установка

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/goodassumption/RequestsProject.git
    cd https://github.com/goodassumption/RequestsProject.git
    ```
2.  Установите необходимые зависимости:
    ```bash
    pip install requests
    ```

## Использование

1.  Запустите приложение из корневой директории репозитория:
    ```bash
    python3 main.py
    ```

2.  Следуйте инструкциям в консоли:
    *   Сначала вам будет предложено выбрать раздел API (посты, комментарии, фотографии, пользователи).
    *   Затем, в зависимости от выбранного раздела, вам предложат выбрать параметр для поиска.
    *   Введите необходимые данные (ID, username и т.д.) для выполнения запроса.
    *   Результаты поиска будут отображены в консоли.
    *   После каждого запроса вы сможете продолжить поиск или завершить работу приложения.

## Структура проекта

Все файлы проекта находятся в корневой директории репозитория.

*   `<ИМЯ_ВАШЕГО_ФАЙЛА_С_КОДОМ>.py`: Основной файл приложения, содержащий всю логику.
*   `logs.txt`: Файл для записи логов.

## Функции

`make_log(log_text='Am I teapot?')`

Добавляет новую запись с временной меткой в лог-файл `logs.txt`.

`check_int(object: str, cur_funk, range = None)`

Проверяет, является ли строка числом и находится ли оно в заданном диапазоне.

`clear()`

Очищает консоль.

`string_search()`

Функция для поиска по строке (в данный момент не полностью реализована).

`choose_page_type(url=default_values.get('CONST_URL'))`

Отображает меню для выбора раздела API.

`choose_parameter(url = default_values.get('CONST_URL'), page_type=default_values.get('page_type'))`

Отображает меню для выбора параметра поиска в зависимости от выбранного раздела.

`make_request(url=default_values.get('url'), page_type=default_values.get('page_type'))`

Выполняет HTTP GET-запрос, обрабатывает ответ и выводит данные в консоль.

`main()`

Основной цикл приложения, управляющий последовательностью вызовов других функций.

`exit_mod(e = SillyException, funk_to_go = None, isException = True)`

Универсальный обработчик исключений и завершения работы программы.

## Зависимости

*   `requests`: Для выполнения HTTP-запросов.
*   `os`: Для работы с операционной системой (очистка консоли).
*   `json`: Для работы с JSON-данными (неявное использование через `requests`).
*   `datetime`: Для получения текущей даты и времени при логировании.
*   `sys`: Для работы с системными параметрами (перенаправление `stdout`).
*   `time`: Для использования задержек (`sleep`).
*   `http_status_descriptions`: Кастомный модуль для описания HTTP-статусов

## Пользовательские исключения

*   `ClientSideError`: Ошибка на стороне клиента.
*   `ServerSideError`: Ошибка на стороне сервера.
*   `SillyException`: Непонятная ошибка.
*   `IncorrectInput`: Неверный ввод.
*   `MyValueError`: Неправильное значение.
*   `TemporaryUnavailable`: Функция в разработке.
*   `NotFound`: Ничего не найдено.
